# Document relations example

Пример последовательно ищет записи по названию документа во всех известных
web-реестрах из каталога `registries.Known()` и выводит JSON со списком
найденных записей и их внутренних связей.

Каждая найденная запись содержит компактное typed-представление:

- `type` - имя подготовленной модели, например `StandardUnit`, `StandardSampleType` или `Method`;
- `data` - поля этой модели без raw-свойств;
- `relations` - связанные записи в таком же typed-формате.

Запуск с выводом в консоль:

```bash
go run ./example -query "ПНД Ф 14.1:2:3:4.48-2022"
```

Сохранение результата в файл:

```bash
go run ./example -query "ПНД Ф 14.1:2:3:4.48-2022" -out /tmp/arshin-related.json
```

Полезные флаги:

- `-registry` - один или несколько реестров через запятую; если не задан,
  используются все известные реестры;
- `-field` - поле для поиска документа, по умолчанию `foei:NameCMM`;
- `-page-size` - размер страницы поиска, по умолчанию `20`;
- `-max-pages` - максимум страниц, которые нужно прочитать, по умолчанию `5`;
- `-timeout` - общий таймаут live-запросов, по умолчанию `2m`.
